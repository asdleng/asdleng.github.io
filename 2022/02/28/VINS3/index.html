VINS-Mono学习笔记3——estimator(上)

发表于 2022-02-28

对vins_estimator的初始化和滑动窗口非线性优化进行了解析

estimator是VINS-Mono的核心模块，其主要内容包括了VINS的初始化、紧耦合的滑动窗口非线性优化、关键帧选择

初始化过程的参数和步骤(按初始化顺序)：

  • 陀螺仪bias
  • SfM每一帧的速度，重力向量，尺度因子
  • Gravity Refifinement，根据这一特点，对重力方向再进行优化
  • 以启动瞬间的位置为原点，重力方向为z轴负向，把所有初始化的参数旋转到该坐标系下，对SfM加尺度因子

滑动窗口非线性优化：

由n个关键帧组成的滑动窗口，根据关键帧选择策略，对其进行添加新的帧，边缘化掉老帧

状态向量：
  • 每一帧图像接收时刻的
  • 外参
  • 所有特征点第一次被观测到时的逆深度值
最大后验估计的残差项：
  • 边缘化残差
  • imu预积分残差
  • 重投影残差

vins_estimator

estimator_node

初始化、滑动窗口的ros节点，在接收特征点和imu数据时进行多线程加锁

函数：
  • process()
    VIO主线程，在接受数据完成后会被唤醒，在提取measurements时互斥锁m_buf会锁住，此时无法接收数据
  • predict()
    从imu测量值imu_msg和上一个PVQ递推得到下一个tmp_Q，tmp_P，tmp_V，中值积分
  • update()
    调用predict()，对imu_buf中所有imu进行PVQ中值积分递推，得到当前帧imu更新项
  • getMeasurements()
    将imu与图像数据进行对齐并组合，返回组合后的测量数据
  • imu_callback()
    imu回调函数，不停地压入imu_buf，在压入时进行加锁
  • feature_callback()
    特征点回调函数，不停地压入feature_buf，在压入时进行加锁
  • restart_callback()
    重定位回调函数

estimator.h/estimator.cpp

里面有estimator类

变量：
  • enum SolverFlag
    求解标志位，INITIAL = 0代表初始化，NON_LINEAR = 1代表初始化成功，开始非线性优化
  • enum MarginalizationFlag
    边缘化标志位，MARGIN_OLD = 0代表边缘化老帧，MARGIN_SECOND_NEW = 1代表边缘化次新帧
  • Vector3d g
    重力向量
  • Matrix3d ric[NUM_OFCAM]
    Vector3d tic[NUM_OF_CAM]
    相机到body(imu)坐标系的旋转矩阵和平移向量
  • Vector3d Ps[(WINDOW_SIZE + 1)]
    Vector3d Vs[(WINDOW_SIZE + 1)]
    Matrix3d Rs[(WINDOW_SIZE + 1)]
    Vector3d Bas[(WINDOW_SIZE + 1)]
    Vector3d Bgs[(WINDOW_SIZE + 1)]
    滑动窗口中的P,V,R,Ba,Bg
  • double td
    imu和图像的时间偏差
  • Matrix3d back_R0
    Vector3d back_P0
    窗口中最老帧的位姿
  • Matrix3d last_R
    Vector3d last_P
    上一帧窗口中最新帧的位姿（故障检测的时候用了）
  • Matrix3d last_R0
    Vector3d last_P0
    上一帧窗口中最老帧的位姿（故障检测的时候用了）
  • std_msgs::Header Headers[(WINDOW_SIZE + 1)]
    用于记录滑动窗口中各帧的时间戳
  • IntegrationBase *pre_integrations[(WINDOW_SIZE + 1)]
    滑动窗口中的预积分类
  • Vector3d acc_0, gyr_0
    imu观测到的原始数据
  • vector<double> dt_buf[(WINDOW_SIZE + 1)]
    vector<Vector3d> linear_acceleration_buf[(WINDOW_SIZE + 1)]
    vector<Vector3d> angular_velocity_buf[(WINDOW_SIZE + 1)]
    窗口中的所有帧的dt,a,v
  • int frame_count
    窗口中已经填入的图像帧的数量
  • int sum_of_outlier 没用 int sum_of_back 边缘一个老帧就+1 int sum_of_front 新来的不是关键帧就+1 int sum_of_invalid 没用
  • FeatureManager f_manager
    特征管理器对象，用于对特征点进行操作
  • MotionEstimator m_estimator
    对极约束对象，用于5点法求解RT
  • InitialEXRotation initial_ex_rotation
    外参标定对象，用于初始化时外参标定
  • bool first_imu
    是否是第一帧imu
  • bool failure_occur
    故障检测
  • vector<Vector3d> key_poses
    关键帧的3D坐标
  • double initial_timestamp
    初始时间戳
  • double para_Pose[WINDOW_SIZE + 1][SIZE_POSE]
    将P、R转化成数组用于Ceres求解，7个数，X,Y,Z,w,x,y,z
  • double para_SpeedBias[WINDOW_SIZE + 1][SIZE_SPEEDBIAS]
    将速度、偏置转化成数组用于Ceres求解，9个数， 3个方向、 3个方向、 3个方向
  • double para_Feature[NUM_OF_F][SIZE_FEATURE]
    将特征点转化成数组用于Ceres求解，第一维为特征点个数，第二维为特征点深度(就一个数弄个二维的?)
  • double para_Ex_Pose[NUM_OF_CAM][SIZE_POSE]
    将外参转化成数组用于Ceres求解，7个数，X,Y,Z,w,x,y,z
  • double para_Td[1][1]
    将时间偏差转化成数组用于Ceres求解
  • MarginalizationInfo *last_marginalization_info
    🤔
  • vector<double *> last_marginalization_parameter_blocks
    🤔
  • map<double, ImageFrame> all_image_frame
    map容器，表示所有图像帧，map<时间戳，图像帧>，图像帧内的信息包括特征点、位姿Rt、预积分对象pre_integration，是否是关键帧
  • IntegrationBase *tmp_pre_integration
    imu预积分对象，用于临时存放imu数据写入pre_integrations
  • bool relocalization_info
    double relo_frame_stamp
    double relo_frame_index
    int relo_frame_local_index
    vector<Vector3d> match_points
    double relo_Pose[SIZE_POSE]
    Matrix3d drift_correct_r
    Vector3d drift_correct_t
    Vector3d prev_relo_t
    Matrix3d prev_relo_r
    Vector3d relo_relative_t
    Quaterniond relo_relative_q
    double relo_relative_yaw
    重定位的变量🤔
函数：
  • processIMU()
    计算加速度角速度，把每一帧的imu测量信息加到imu预积分实例数组pre_itergrations[]里，同时得到每一帧的
  • processImage()
    初始化，判断是否关键帧，进行外参标定，滑动窗口非线性优化，调用的函数包括：
  • FeatureManager::addFeatureCheckParallax()
  • FeatureManager::getFeatureCount()
  • FeatureManager::getCorresponding()
  • Estimator::solveOdometry()
  • Estimator::slideWindow()
  • FeatureManager::removeFailures()

  • Estimator::initialStructure()

  • initialStructure()
    视觉结构初始化，用方差判断imu激励是否足够，由基础矩阵得到的R、t作为初始值，恢复滑动窗口帧的位姿，然后进行视觉惯性联合初始化，调用的函数包括：
  • Estimator::relativePose()
  • GlobalSFM::construct()

  • Estimator::visualInitialAlign()

  • visualInitialAlign()
    得到尺度s和重力g后，更新所有图像帧在世界坐标系下的Ps、Rs、Vs
  • relativePose()
    判断两帧有足够视差30且内点数目大于12则可进行初始化，同时得到和
  • solveOdometry()
    三角化所有特征点深度，并进行非线性优化，调用的函数有：
  • FeatureManager::trangulate()

  • Estimator::optimization()

  • optimization()
    非线性优化，核心函数
  • slideWindow()
    实现滑动窗口，如果次新帧是关键帧，则边缘化最老帧，如果次新帧不是关键帧，则舍弃次新帧视觉保留imu
  • slideWindowOld()
    滑动窗口边缘化最老帧时处理特征点被观测的帧号
  • slideWindowNew()
    滑动窗口边缘化次新帧时处理特征点被观测的帧号
  • vector2double()
    vector转换成double数组，因为Ceres使用数值数组，Ps、Rs转变成para_Pose，Vs、Bas、Bgs转变成para_SpeedBias
  • double2vector()
    数据转换，vector2double的相反过程，同时这里为防止优化结果往零空间变化，会根据优化前后第一帧的位姿差进行修正
  • failureDetection()
    系统故障检测，特征太少、偏置过大、旋转过大

  • setReloFrame()
    重定位

feature_manager.h/feature_manager.cpp

里面有FeaturePerFrame类、FeaturePerId类、FeatureManager类

class FeaturePerFrame

每个特征点在一张图像中的信息

变量：
  • cur_td
    当前帧imu相机时间差
  • Vector3d point
    当前帧相机坐标系下特征点归一化3d坐标
  • Vector2d uv
    当前帧特征点像素坐标
  • Vector2d velocity
    当前帧特征点像素的速度

class FeaturePerId

特征类，每个特征点由多个连续图像观测到

变量：
  • const int feature_id
    特征点的id索引
  • int start_frame
    该特征点第一次被追踪的帧在滑动窗口中的id
  • vector<FeaturePerFrame> feature_per_frame
    特征点被哪些帧追踪到的vector容器
  • int used_num
    该特征点被追踪到的次数
  • bool is_outlier
    是否是异常点
  • estimated_depth
    该特征点被估计的逆深度
  • solve_flag
    逆深度估计是否成功
函数：
  • endFrame()
    返回该特征点丢失时的帧在滑动窗口中的id

class FeatureManager

特征管理器类，里面有list容器管理所有特征

变量：
  • list<FeaturePerId> feature
    滑动窗口内所有特征点
  • int last_track_num
    上一帧图像追踪到的特征点个数
  • const Matrix3d *Rs
    滑动窗口中所有图像帧的旋转矩阵
函数：
  • setRic()
    设置 ric
  • clearState()
    清空
  • getFeatureCount()
    返回窗口中特征点总个数
  • addFeatureCheckParallax()
    把特征点放入feature的list容器中，计算每一个点跟踪次数和它在次新帧和次次新帧间的视差，返回是否是关键帧
  • getCorresponding()
    返回两帧之间对应的特征点的归一化3d点pair坐标
  • setDepth()
    设置所有特征点逆深度值
  • removeFailures()
    删除估计逆深度失败的点
  • clearDepth()
  • getDepthVector
  • triangulate()
    对特征点进行三角化求深度（SVD分解）对应VIO课前端三角化一节
  • removeBackShiftDepth()
    边缘化最老帧时，处理特征点保存的帧号，将起始帧是最老帧的特征点的深度值进行转移
  • removeBack()
    边缘化最老帧时，直接将特征点所保存的帧号向前滑动
  • removeFront()
    边缘化次新帧时，对特征点在次新帧的信息进行移除处理
  • removeOutlier()
    移除异常点
